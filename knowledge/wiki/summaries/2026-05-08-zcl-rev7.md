## 概述

ZCL Revision 7 是 Zigbee Cluster Library 的 2018 版本，定义标准 Cluster 的属性、命令、数据类型和行为，是 Revision 8 的直接前版本。本次重新 ingest 不是只建立来源占位，而是按目录、Foundation、主要功能域和关键 Cluster 分段精读后更新的版本。

### Foundation 精读结论

- ZCL 是 Cluster 功能的仓库，开发新应用时应优先复用 ZCL cluster，而不是重新发明相同接口。Cluster specification 定义独立功能实体，应用 cluster 应尽量不依赖应用域之外的功能；utility cluster 可以提供到其他层的接口，例如 Groups 到 group addressing。
- Cluster identifier 是 16-bit 值，一个 identifier 映射到一个 cluster specification；一个抽象 specification 可以被多个更具体的 identifier 映射。Rev 7 明确支持 base/derived cluster 模型和 instance model：同一 endpoint 同时支持 base server cluster id 与 derived server cluster id 时，两者代表同一个实例并保持兼容。
- Conformance 用 Mandatory、Optional、Deprecated 表达，但强制性通常受 server/client、cluster id 是否可发现、可选属性依赖等条件约束。Deprecated 属性/命令需要标注并删除描述正文。
- Client/server 模型中，通常保存属性的一侧是 server，操纵属性的一侧是 client；simple descriptor 的 input cluster list 表示 server cluster，output cluster list 表示 client cluster。
- Foundation 定义的全局命令覆盖 Read/Write Attributes、Configure/Read Reporting、Report Attributes、Default Response、Discover Attributes/Commands、Structured Read/Write 等。实现 attributes 的 cluster 通常必须支持 read/write/discover；reporting 相关命令只在存在 mandatory reportable attributes 时强制。
- ZCL frame 由 frame control、可选 manufacturer code、transaction sequence number、command identifier 和 payload 组成。Frame control 包含 frame type、manufacturer specific、direction、disable default response 等字段。
- Reporting 的 direction=0x00 用于配置发送报告，direction=0x01 用于配置接收报告超时。离散类型变化即触发报告，模拟量类型按 reportable change 触发；minimum interval 抑制频繁报告，maximum interval 控制周期报告或关闭/恢复默认配置。
- Structured attributes 对 Array/Structure/Set/Bag 引入 selector。数组/结构元素从 1 开始编号，0 号元素表示元素数量；Set/Bag 支持整集合写入、按值 add/remove，Set 不允许重复值，Bag 允许重复值。
- Addressing 和类型空间边界清晰：standard profile `0x0000..0x7fff`，manufacturer-specific profile `0xc000..0xffff`；standard cluster `0x0000..0x7fff`，manufacturer-specific cluster `0xfc00..0xffff`；standard attribute `0x0000..0x4fff`，global attribute `0xf000..0xfffe`。

### 重点机制

- On/Off cluster Rev 7 包含 `OnOff`, `GlobalSceneControl`, `OnTime`, `OffWaitTime`, `StartUpOnOff`，命令包括 mandatory `Off`, `On`, `Toggle` 和 optional `Off with effect`, `On with recall global scene`, `On with timed off`。见 [[entities/cluster-on-off]]。
- Poll Control cluster `0x0020` 是 ZCL 层管理 end device polling 的机制，server 位于 end device，client 可在 check-in 后让设备进入 short poll/fast poll。它引用 MAC Data Request/parent 关系，但不是 NWK/MAC 层 broadcast 机制本身。见 [[concepts/sleepy-end-device-broadcast]]。
- OTA Upgrade cluster `0x0019` 采用 client polling 为基本机制，ZED/ZR clients 都周期性 query server；server 的 Image Notify 是可选能力。OTA 安全分为 image verification、transport 和 encryption，是否要求签名/加密由应用标准决定。
- Commissioning cluster `0x0015` 操作 Zigbee stack startup/join/end-device/concentrator 参数，规范特别强调授权和安全处理；Touchlink `0x1000` 分为 inter-PAN 的 touchlink command set 和 standard unicast 的 commissioning utility command set。
- Smart Energy 章节中 Price、DRLC、Metering、Messaging、Tunneling、Key Establishment 面向 SE 应用，但文档也说明这些 cluster 可用于其他应用域，且常伴随应用层安全约束。

## 关键要点

- Rev 7 是一个 15 章的组合规范，不是单一 Markdown 式文档；后续引用应带章节/页码。
- Foundation 是实现互操作性的核心：frame、global command、reporting、data type、status enum、manufacturer-specific extension 都从这里派生。
- Cluster 定义和测试规范必须分开：本文件给协议/cluster 行为，`raw/test-specs/...` 给 certification 测试步骤。
- Rev 7 可作为 Rev 8 对比和旧设备/旧测试规范追溯基准，但新开发应明确比较 Rev 8 或更高版本差异后再下结论。
